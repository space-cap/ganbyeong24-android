# 시니어 친화적 간병 신청 UI 개선

## 📋 개요
현재 간병 신청 화면은 8개의 입력 필드를 한 화면에 모두 표시하고 있어, 나이 많은 사용자(시니어)가 사용하기 어렵습니다. 3단계 방식으로 나누고, 텍스트 입력을 최소화하여 사용성을 개선합니다.

## 🎯 목표
- 입력 과정을 3단계로 분리하여 인지 부담 감소
- 텍스트 입력 대신 버튼 선택 및 DatePicker 활용
- 큰 폰트, 넓은 터치 영역으로 시니어 친화적 UI 구현

## 📝 상세 작업 내용

### 1. 공통 컴포넌트 생성
- [ ] `StepIndicator.kt` - 1/2/3 단계 진행 상황 표시 컴포넌트
- [ ] `PatientConditionSelector.kt` - 환자 상태 선택 버튼 (거동 가능/불가/휠체어/와상)
- [ ] `DatePickerField.kt` - Material3 DatePicker를 활용한 날짜 선택 필드

### 2. State 및 ViewModel 수정
- [ ] `CareRequestState.kt`에 `currentStep: Int` 필드 추가
- [ ] `CareRequestViewModel.kt`에 단계 네비게이션 로직 추가
  - `nextStep()` - 현재 단계 유효성 검사 후 다음 단계로 이동
  - `previousStep()` - 이전 단계로 이동
  - `validateStep1()`, `validateStep2()`, `validateStep3()` - 단계별 유효성 검사

### 3. 화면 재구성 (3단계)
- [ ] **Step 1: 환자 정보**
  - 환자명 입력 (TextField)
  - 환자 상태 선택 (버튼 4개: 거동 가능/거동 불가/휠체어 사용/와상 환자)
  
- [ ] **Step 2: 간병 기간**
  - 간병 시작일 (DatePicker)
  - 간병 종료일 (DatePicker)
  
- [ ] **Step 3: 연락처 및 위치**
  - 보호자명 입력
  - 보호자 연락처 입력
  - 위치 입력
  - 환자 연락처 입력 (선택사항)

### 4. UI 개선 사항
- [ ] 폰트 크기: 20-24sp (기존 16sp에서 증가)
- [ ] 버튼 높이: 56dp 이상
- [ ] 선택 카드 높이: 80dp
- [ ] 패딩: 32dp (기존 24dp에서 증가)
- [ ] 항목 간 간격: 24dp
- [ ] 이전/다음 버튼 추가 (마지막 단계에서는 "신청하기")

## ✅ 완료 조건
- [ ] 3단계 UI가 정상적으로 동작
- [ ] 각 단계별 유효성 검사 통과 시에만 다음 단계로 이동
- [ ] 이전 버튼으로 이전 단계 이동 가능
- [ ] DatePicker로 날짜 선택 가능
- [ ] 환자 상태를 버튼으로 선택 가능
- [ ] 빌드 성공 및 Preview 정상 동작
