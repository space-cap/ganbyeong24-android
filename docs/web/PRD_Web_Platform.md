# 간병24 웹 플랫폼 프로젝트 기획서 (PRD)

## 📋 프로젝트 개요

### 목적
간병24 모바일 앱의 기능을 웹으로 확장하여 접근성을 높이고, 관리자 페이지를 통해 효율적인 매칭 서비스를 제공합니다.

### 배경
- 모바일 앱만으로는 시니어층의 접근성 제한
- 매칭 담당자가 신청 내역을 확인하고 전화 상담을 진행할 수 있는 시스템 필요
- 웹 검색을 통한 유입 경로 확보 필요

### 목표
1. **간병 신청 웹페이지**: 누구나 쉽게 간병 서비스를 신청할 수 있는 공개 웹사이트
2. **관리자 대시보드**: Firebase 데이터를 실시간으로 조회하고 관리할 수 있는 내부 시스템

---

## 🎯 핵심 기능

### 1. 간병 신청 웹페이지 (Public)

#### 1.1 홈페이지
- **서비스 소개**: 간병24가 제공하는 서비스 설명
- **주요 기능 안내**: 간병 신청, 간병사 등록 프로세스 설명
- **CTA 버튼**: "간병 신청하기", "간병사 등록하기"
- **앱 다운로드 링크**: Google Play Store 링크

#### 1.2 간병 신청 폼
**필수 입력 정보** (앱과 동일):
- 환자 이름
- 환자 나이
- 환자 성별
- 병원 이름
- 병실 정보
- 간병 시작일
- 간병 기간
- 보호자 연락처
- 특이사항 (선택)

**기능**:
- 실시간 입력 검증
- 시니어 친화적 UI (큰 글씨, 명확한 버튼)
- Firebase Firestore에 데이터 저장
- 제출 완료 후 안내 메시지

#### 1.3 간병사 등록 폼
**필수 입력 정보** (앱과 동일):
- 이름
- 연락처
- 경력 (년)
- 자격증 보유 여부
- 가능 지역
- 희망 급여
- 자기소개

**기능**:
- Firebase Firestore에 데이터 저장
- 제출 완료 후 안내 메시지

#### 1.4 SEO 최적화
- 메타 태그 설정
- 구조화된 데이터 (Schema.org)
- 모바일 반응형 디자인
- 빠른 로딩 속도

---

### 2. 관리자 대시보드 (Admin)

#### 2.1 인증 시스템
- **Firebase Authentication** 사용
- 관리자 계정만 접근 가능
- 이메일/비밀번호 로그인

#### 2.2 대시보드 홈
**주요 지표**:
- 오늘의 간병 신청 건수
- 오늘의 간병사 등록 건수
- 전체 누적 통계
- 최근 7일 트렌드 그래프

#### 2.3 간병 신청 관리
**목록 화면**:
- 모든 간병 신청 내역 조회 (Firebase Firestore 실시간 연동)
- 필터링: 날짜, 병원, 상태
- 정렬: 최신순, 시작일순
- 검색: 환자명, 보호자 연락처

**상세 화면**:
- 신청 정보 전체 표시
- 보호자 연락처 **원클릭 전화** 버튼
- 메모 기능 (상담 내용 기록)
- 상태 변경: 대기중 → 상담중 → 매칭완료 → 취소

#### 2.4 간병사 관리
**목록 화면**:
- 모든 간병사 등록 내역 조회
- 필터링: 지역, 경력, 자격증 유무
- 정렬: 최신순, 경력순
- 검색: 이름, 연락처

**상세 화면**:
- 간병사 정보 전체 표시
- 연락처 **원클릭 전화** 버튼
- 메모 기능
- 상태 변경: 대기중 → 활동중 → 휴면

#### 2.5 매칭 관리 (향후 확장)
- 간병 신청과 간병사를 수동으로 매칭
- 매칭 이력 관리
- 매칭 성공률 통계

## 🛠 기술 스택

### Frontend

#### 공통 스택 (간병 신청 웹페이지 + 관리자 대시보드)
- **Framework**: React 18 + Vite 5
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **UI Components**: shadcn/ui
- **Routing**: React Router v6
- **Form Validation**: React Hook Form + Zod
- **State Management**: Zustand (또는 React Context API)
- **Charts**: Recharts (관리자 대시보드)
- **Table**: TanStack Table (관리자 대시보드)

**선택 이유:**
- ✅ 빠른 개발 속도 (Vite의 초고속 HMR)
- ✅ 단순한 프로젝트 구조
- ✅ 가벼운 번들 사이즈
- ✅ 익숙한 개발 환경

### Backend
- **Database**: Firebase Firestore (기존 앱과 동일)
- **Authentication**: Firebase Authentication
- **Hosting**: Vercel / Netlify / Firebase Hosting
- **Functions**: Firebase Cloud Functions (필요시)

### 공통
- **Real-time Updates**: Firebase Firestore 실시간 리스너
- **Analytics**: Google Analytics 4
- **Build Tool**: Vite 5
